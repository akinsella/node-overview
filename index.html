Failfast et résilience

  • Gestion des erreurs spartiate. Si une erreur n'est pas catchée, alors le process tombe
  • Difficulté à catcher les erreurs
  • Outil de monitoring de processus obligatoire

Domaines

  • Le concept de domaine aide node.js à être plus résilient face aux erreurs
  • Les domaines permettent d’exécuter un callback dans un contexte d’exécution où tous les EventEmitter et les exceptions seront interceptées et rattachées au domaine.
  • Le domaine est lui-même un EventEmitter qui va donc émettre un évènement error quand il a intercepté une erreur.

                domain.create()
                   .on('error', function (err) {
                        // handle error
                    })
                    .run(hello);

Clusterisation

  • Node propose une API de clusterisation, permettant de multiples instances du programmes
  • Idéalement, une instance par cpu
  • La clusterisation ne supprime par le besoin de monitoring des process
  • L'API de clusterisation est considérée comme instable par la documentation node.js
  • Dans les faits l'API est stable...
